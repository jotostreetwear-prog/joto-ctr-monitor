# JOTO — Заказы на производство (приложение Битрикс24)

Отдельный сервис: добавляет в **левое меню** Битрикс24 пункт **«Заказы на производство»**.

## Как работает

1. Сотрудник открывает пункт меню и создаёт заказ:
   **наименование, изготовитель, размеры, цвет, количество, стоимость, сроки** + прикрепляет файл.
2. Заказ уходит **на согласование Алану** (ему приходит уведомление в Битрикс).
3. Алан **согласовывает** или отклоняет заказ прямо в приложении.
4. После согласования автоматически:
   - создаётся **задача для Марины Ваниной** (с описанием заказа и прикреплённым файлом);
   - создаётся **сделка в CRM**.

## Деплой на Railway (отдельный сервис)

1. Создайте на Railway **новый проект** (отдельно от CTR-монитора) из этого репозитория.
2. Добавьте плагин **PostgreSQL** — переменная `DATABASE_URL` подставится сама
   (без неё данные сбрасываются при передеплое).
3. Задайте переменные окружения:

| Переменная | Значение |
|---|---|
| `B24_CLIENT_ID` | client_id приложения из Битрикс24 |
| `B24_CLIENT_SECRET` | client_secret приложения из Битрикс24 |
| `APP_BASE_URL` | публичный адрес сервиса, напр. `https://joto-production-orders.up.railway.app` |
| `ALAN_USER_ID` | ID Алана в Битрикс24 (кто согласовывает) |
| `MARINA_USER_ID` | ID Марины Ваниной (кому падает задача) |
| `DEAL_CATEGORY_ID` | (необязательно) ID воронки CRM для сделок |
| `SECRET_KEY` | любая случайная строка |

> ID сотрудников можно узнать прямо в приложении — кнопка **«ID сотрудников»**.

## Настройка приложения в Битрикс24

Разработчикам → Другое → **Локальное приложение** (серверное):

- **Путь обработчика**: `https://<APP_BASE_URL>/app`
- **Путь для первоначальной установки**: `https://<APP_BASE_URL>/install`
- Отметить «Использует только API» — нет (нужен интерфейс).
- **Права (scope)**: `placement`, `im`, `tasks`, `crm`, `disk`, `user`.

После сохранения скопируйте **client_id / client_secret** в переменные Railway и нажмите
**«Переустановить приложение»** — в левом меню появится пункт «Заказы на производство».

## Локальный запуск

```bash
pip install -r requirements.txt
python main.py
```
