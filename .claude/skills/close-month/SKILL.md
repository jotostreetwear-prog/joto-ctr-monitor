# Close Month

Проводит продавца через закрытие месяца: собрать всё, сверить, посчитать прибыль, подготовить данные дальше (учёт/налоги). Цель — чтобы цифры сошлись и ничего не потерялось.

## Когда применять

«Закрываю месяц», «подбей итоги за месяц», «сверка за период», «сколько заработал в этом месяце», «подготовь данные для бухгалтера».

## Чек-лист закрытия

Идти по порядку, по каждому пункту фиксировать статус (готово / расхождение / нет данных):

1. **Отчёты о реализации** со всех площадок за месяц собраны и суммы сходятся с выплатами.
2. **Удержания** разнесены по статьям: комиссия, логистика, хранение, реклама, штрафы.
3. **Возвраты и компенсации** учтены (и в выручке, и в логистике).
4. **Остатки товара** на конец месяца зафиксированы (для себестоимости проданного).
5. **Расходы вне площадки**: закупка, ФОТ, услуги, подписки, налоги.
6. **Эквайринг и прочие комиссии** учтены.
7. **Сверка кэша**: фактические поступления = выплаты по отчётам (используй скил `cash-flow-snapshot`).

## Расчёт прибыли за месяц

```
Выручка (по отгрузке/реализации)
− Себестоимость проданного товара
− Комиссии и удержания площадки
− Возвраты
− Реклама и продвижение
− Прочие операционные расходы
− Налог
= Прибыль за месяц
```

Сравни с прошлым месяцем, если данные есть.

## Формат вывода

1. Чек-лист со статусами (что закрыто, где расхождения).
2. P&L за месяц (таблица; при необходимости — выгрузка через скил `xlsx`).
3. Список открытых вопросов / расхождений, которые нужно добить до сдачи бухгалтеру.

## Важно

Закрытие не считается завершённым, пока есть расхождения между отчётами площадки и фактическими поступлениями. Подсвечивай такие расхождения отдельно, а не прячь в итог.
